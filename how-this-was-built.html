How this site was built

People keep asking who built this site. The honest answer: I described what I wanted, in plain English, to an AI coding assistant — and corrected it until it looked right. No bootcamp, no $5k freelancer, no framework. An afternoon and a lot of opinions.

Here's the entire recipe — the tools, and the actual kind of prompts I used — so you can build your own. The whole thing is open on GitHub; feel free to steal it.

01

The ingredients

Four things, all of them cheap or free:

  • An AI coding assistant. The pair that actually reads and writes the files for you. I used Claude Code — you tell it what you want, it edits the page.
  • A GitHub repo. A home for the files that remembers every change, so nothing is ever truly broken — you can always undo.
  • Plain HTML & CSS — no framework. The raw building blocks every web page is made of. Two text files. That's the whole site you're reading.
  • A small server + Caddy. A ~€4/month box (Hetzner) running Caddy, which serves the page and sets up HTTPS automatically. Skip this and use any free host if you prefer.

You do not need to know how to code. You need to know what you want — and be willing to say "no, not like that."

02

The recipe

  1. Describe the vibe, not the code

    Start with a feeling and a structure. Don't worry about how — that's the assistant's job. Mine looked roughly like this:

    Build me a one-page personal website. Warm, minimal, lots of whitespace, one accent colour, a typographic feel — deliberately not generic-AI-looking.

    Sections: a hero with my one-liner, About, what I work on, writing, talks, and contact. Plain HTML and CSS, no framework. Make it responsive and support a dark mode.

  2. Let it build the first draft

    Then just let it run. In seconds you have a real page you can open in your browser.

    Go ahead and create the files. Then tell me how to preview it locally.

    It won't be perfect. That is entirely the point — you now have something to react to, which is much easier than starting from a blank page.

  3. Point and correct — this is 80% of it

    Now you art-direct. Look at the page, find the thing that bugs you, and say so in one specific sentence. Repeat. These were real nudges:

    The headline is too big on mobile and the photo looks squished — fix both.

    Strip it back — make it near-monochrome, one clean typeface, more whitespace.

    Mobile feels like a lot of scrolling. Consult UX and brand experts, then fix it.

    Add a little coffee cup in the corner that fills up as you scroll. 🙂

    You're not coding — you're being picky out loud. (Yes, the cup down there is real.)

  4. Pour in your real content

    Hand over the facts in your own words and let it handle the markup. Paste, don't fuss about formatting.

    Replace the placeholder text with this: [paste your bio, your links, your projects]. Here are my photos — add them to the About gallery.

    Keep the words yours. Let the assistant worry about tags, alt text and accessibility.

  5. Don't skip the unglamorous bits

    The assistant is just as happy doing the parts people usually avoid — legal pages, privacy, performance, being findable on Google. Ask for them by name:

    Add an imprint and a privacy page for my country, self-host the font so the site doesn't phone home, and set it up so Google can index it.

  6. Put it on the internet

    Point your domain at a server, push your files, and let Caddy fetch a free HTTPS certificate. You can ask the assistant to write that config for you too:

    Write the web-server config to serve this folder at mydomain.com with automatic HTTPS, and walk me through deploying it.

    No domain or server yet? Any free static host works just as well to start.

That's the whole trick: describe what you want, be honest about what's wrong, repeat. If you can picture it and you're a little stubborn about the result, you can have a site like this by tonight. The source is right here on GitHub — go make it yours. ☕
